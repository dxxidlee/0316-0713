## Overview

A personal archive database site. Minimal, archival aesthetic. White background, cyan (#00B4D8 approx) as the only accent color. Typography: PP Neue Montreal throughout. All entries live in entries.json and are rendered dynamically.

## Layout

### Header (top bar)

Split into four zones across the full width:

**Zone 1 — top left:**
- `(0316*0713)®` in bold, large
- Cyan filled rectangle below it (decorative block)
- `0316*0713 Archive™` in cyan below that

**Zone 2 — top center-left:**
- Label: `Search Archive:`
- Text input below it (searches title + source live)

**Zone 3 — top center-right:**
- Dropdown: `Default` (selected by default)
- Below it a stacked list of filter options:
  `Title / Date Added / Date Created / Text / Audio / Photo/Video`
- This is a filter — selecting one filters the table by type or resets to default sort

**Zone 4 — top right:**
- `Last Modified: [date and time pulled from entries.json or set manually]`

### Right Sidebar (fixed, vertical strip on far right edge)

Five icon buttons stacked vertically:
1. Hamburger / list view (default active state)
2. Single square / 1-column grid
3. Two columns / 2-col grid
4. Three columns / 3-col grid
5. Four columns / 4-col grid
6. ⓘ info button (green, bottom) — opens info overlay

Note: grid views are lower priority, implement list view fully first. Grid views are most useful for photo/video entries.

### Table (main content area)

Column headers: `# / Title / Source / Year / Type`

Each row: one entry. Columns match headers.

Row numbers are zero-padded: 01, 02, 03...

Type label displayed as plain text (Text, Audio, Video, Photo).

Thin horizontal rules between every row.

No row hover background — keep it clean. Cursor pointer only.

Active/expanded row: title, source, year, type all turn cyan. Row number gets a downward arrow indicator below it.

## Interactions

### Filter (dropdown)

Options: Default, Title, Date Added, Date Created, Text, Audio, Photo/Video

- Default: original JSON order
- Title: alphabetical sort
- Date Added / Date Created: chronological sort by those fields
- Text / Audio / Photo/Video: filter table to show only that type
- Selecting a filter re-renders the table immediately
- Row numbers re-index from 01 on filter

### Search

- Live search as user types
- Searches against: title, source
- Works on top of active filter
- Clears when input is emptied

### Expand — Text entries

Click a text row → row expands inline directly below the clicked row. The rest of the table continues below the expanded section.

Expanded section shows:
- Full description/body text
- URL as a clickable link if present
- Down arrow on the row number column

Click the row again to collapse.

### Expand — Audio entries

Click an audio row → row expands inline with:
- Album art thumbnail (square, left side) — if no thumbnail, show a placeholder block
- Custom audio player (center/right):
  - Skip back button | Play/Pause button | Skip forward button
  - Scrubber bar (cyan filled progress, gray track)
  - Current timestamp left, total duration right
- Controls and progress bar in cyan
- Short note/description below if present

### Expand — Video entries

Click a video row → row expands inline with:
- Video player embedded directly in the table row
- Custom controls overlaid on video:
  - Skip back | Play/Pause | Skip forward
  - Scrubber with cyan progress
  - Timestamps bottom left and right
- Caption below if present
- Video does NOT open in a lightbox — it plays inline

### Lightbox — Photo entries

Click a photo row → full-screen lightbox overlay:
- Background: blurred version of the page behind it (backdrop-filter: blur)
- Image centered, large, with rounded corners and a thin cyan border/glow
- Caption below image if present
- Click outside image or press ESC to close
- If multiple photos per entry (future): prev/next arrow navigation

### Info Overlay (⓪ button)

Click the green ⓘ button in the right sidebar:
- Full-screen overlay, background blurred
- Text content shown centered on top
- This is the "about this archive" page — write whatever you want here
- ESC or click outside to close

## Build Order

Follow this sequence exactly — do not skip ahead:

**Step 1 — Font + CSS reset**
Set up @font-face, CSS variables (cyan, black, white, grays), reset. Confirm font renders correctly in browser before moving on.

**Step 2 — Static layout shell**
Hard-code the full header, sidebar, table headers, and 3-4 static rows in HTML. No JS yet. Get all spacing, proportions, and typography pixel-perfect against the design. This is the most important step — get it right before wiring data.

**Step 3 — Load entries.json + render loop**
fetch() entries.json in main.js. Write renderTable() that maps entries to rows. Confirm all entries display with correct columns. Implement row number zero-padding.

**Step 4 — Filter**
Wire the dropdown. Implement filter logic for each option. Re-render table on change. Re-index row numbers.

**Step 5 — Search**
Wire the search input. Live filter on input event. Combine with active filter state.

**Step 6 — Text expand**
Click handler on text rows. Toggle inline expansion with description and link. Cyan active state on row. Down arrow on row number.

**Step 7 — Audio expand**
Click handler on audio rows. Inline player with album art. Custom controls using the HTML audio element with custom UI (do not use browser default controls). Cyan scrubber.

**Step 8 — Video expand**
Click handler on video rows. Inline video player with custom controls. Matches audio player style.

**Step 9 — Photo lightbox**
Click handler on photo rows. Full-screen overlay with blur. Rounded image with cyan border. ESC to close.

**Step 10 — Info overlay**
Wire the ⓘ button. Same overlay pattern as photo lightbox but with text content.

**Step 11 — Polish pass**
Last Modified display. Consistent hover states. Transition/animation on expand (smooth height, not jarring). Full spacing and alignment pass against Figma.

**Step 12 — Vercel deploy**
Connect GitHub repo to Vercel. Auto-deploy on push. No config needed — Vite is auto-detected.

## Notes

- No subpages. Everything happens on one page.
- Only one row can be expanded at a time. Opening a new row collapses the previous one.
- Grid view (sidebar icons 2-5) is lower priority — implement last. List view is the default and most important.
- The sort dropdown doubles as a type filter — it is not two separate controls.
- All media files live in /public/media/ and are referenced by path in entries.json.
- Font files are case-sensitive on Vercel. Match filenames exactly.
